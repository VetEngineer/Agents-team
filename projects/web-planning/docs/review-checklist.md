# Codex PR Review Checklist (Strict)

## 목적
- 버그/리스크/회귀/테스트 누락을 최우선으로 식별
- 코드 스타일보다 동작 안전성 우선

## 체크리스트
- [ ] 요구사항 충족 여부(스펙 대비 누락 없음)
- [ ] 데이터 모델/마이그레이션 변경의 부작용 검토
- [ ] 권한/보안(RLS, 인증/인가) 누락 여부
- [ ] 입력 검증/오류 처리/엣지 케이스 처리
- [ ] 성능/비용 리스크(과도한 쿼리/외부 호출)
- [ ] 회귀 가능성(기존 기능 영향)
- [ ] 테스트 추가/수정 필요 여부
- [ ] 로그/관측성(실패 원인 추적 가능성)
- [ ] 문서 업데이트 필요 여부

## 리뷰 코멘트 원칙
- 심각도 우선순위(P0/P1/P2) 표시
- 파일/라인 기준으로 지적
- 수정 가이드 제시

## 이슈 생성 기준
- 보안/데이터 손실/시스템 다운 위험은 무조건 Issue
- 테스트 누락이 크면 Issue
- 설계적 결함(확장 불가 구조) 발견 시 Issue

---

메모: 이 체크리스트는 프로젝트 진행에 따라 업데이트합니다.
