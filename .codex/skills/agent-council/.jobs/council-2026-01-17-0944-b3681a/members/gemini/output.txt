블로그 자동화 웹앱 구축을 위한 기획 및 개발 로드맵을 제안해 드립니다. 사용자의 메모리에 저장된 선호도(shadcn/ui, 깔끔하고 모던한 디자인)를 반영하여 구상했습니다.

### 1. 핵심 기능 (Core Features)

블로그 운영의 번거로움을 줄이고 콘텐츠 품질을 유지하는 데 초점을 맞춥니다.

*   **트렌드/키워드 발굴:** 구글 트렌드, 네이버 검색어 등을 분석하여 글감 추천.
*   **AI 콘텐츠 생성:**
    *   주제 입력 시 제목, 목차, 본문(SEO 최적화) 자동 생성.
    *   Gemini/GPT를 활용한 톤앤매너(친근함, 전문적 등) 조정.
*   **이미지 자동 생성/매칭:** 본문 내용에 맞는 AI 이미지 생성(DALL-E 3 등) 또는 Unsplash 무료 이미지 매칭.
*   **스마트 에디터:** 생성된 초안을 수정하고 마크다운/HTML로 변환하는 에디터.
*   **자동 게시 및 예약:** WordPress, Tistory, Medium 등 주요 플랫폼 API 연동을 통한 원클릭/예약 발행.
*   **성과 분석 대시보드:** 조회수, 방문자 유입 경로 등 통합 분석 (MVP 이후).

### 2. 기술 스택 (Tech Stack)

빠른 개발과 확장성, 그리고 사용자가 선호하는 모던 UI를 고려했습니다.

*   **Frontend & Full-stack Framework:** **Next.js (App Router)**
    *   SEO에 유리하고 백엔드 로직(API Routes)을 함께 처리하기 좋습니다.
*   **UI Library:** **Tailwind CSS + shadcn/ui**
    *   깔끔하고 'AI스럽지 않은' 정제된 디자인을 위해 필수적입니다. Lucide React 아이콘 활용.
*   **Database:** **Supabase (PostgreSQL)**
    *   인증(Auth), 데이터베이스, 실시간 기능을 한 번에 해결합니다.
*   **AI Engine:** **Google Gemini API** (또는 OpenAI GPT-4)
    *   텍스트 생성 및 멀티모달 분석 용도.
*   **State Management:** Zustand (가볍고 직관적).
*   **Deployment:** Vercel (Next.js 최적화).

### 3. MVP (Minimum Viable Product) 범위

4주 내에 '실제 사용 가능한' 수준을 목표로 합니다.

1.  **회원가입/로그인:** Supabase Auth (Google 로그인).
2.  **글 생성기:** 키워드 입력 -> AI가 글 초안(제목+본문) 생성.
3.  **기본 에디터:** 생성된 글을 수정할 수 있는 심플한 텍스트 에디터.
4.  **단일 플랫폼 연동:** 가장 범용적인 **WordPress** (또는 Tistory) API 연동하여 '발행' 버튼 클릭 시 실제 블로그에 업로드.

### 4. 리스크 및 주의점

*   **저품질 콘텐츠 인식:** AI가 쓴 글을 그대로 올리면 검색 엔진(SEO)에서 스팸으로 분류될 위험이 큽니다. "Human-in-the-loop"(사람의 검수) 과정을 반드시 UX에 녹여내야 합니다.
*   **API 비용:** 사용량이 늘어날 경우 LLM API 비용이 발생합니다. 토큰 관리 및 캐싱 전략이 필요합니다.
*   **플랫폼 정책 변화:** Tistory나 네이버 블로그 등은 API 정책이 폐쇄적이거나 자주 바뀔 수 있습니다. (Selenium 같은 자동화 툴이 필요할 수도 있음)

### 5. 개발 일정 (예상: 4주)

*   **1주차: 기획 및 세팅**
    *   요구사항 구체화, UI/UX 와이어프레임.
    *   Next.js + shadcn/ui 프로젝트 스캐폴딩.
    *   Supabase DB 스키마 설계.
*   **2주차: AI 코어 개발**
    *   Gemini/OpenAI API 연동.
    *   프롬프트 엔지니어링 (양질의 블로그 글 포맷팅).
    *   글 생성 로직 구현.
*   **3주차: 에디터 및 연동**
    *   Tiptap 또는 유사 라이브러리로 에디터 구현.
    *   WordPress API 연동 및 발행 테스트.
*   **4주차: UI 폴리싱 및 배포**
    *   shadcn/ui 기반 디자인 디테일 잡기.
    *   버그 수정 및 Vercel 배포.
